# Architecture

How this portfolio site is built. For a higher-level overview see the [README](../README.md).

## Overview

This is a **static-first Next.js 16 (App Router) site**. Most pages are prerendered at build time; only a few serverless routes run on demand (IndexNow, and the auto-generated SEO/LLM files). There is no database, no authentication, and no newsletter.

```
Browser ──► Next.js (Vercel)
              ├── Static pages (home, about, blog, author, legal)
              ├── SSG blog posts (generated from /content/blog/*.mdx)
              └── On-demand routes (feed.xml, llms.txt, sitemap.xml, /api/indexnow)
```

## Rendering strategy

| Route | Strategy |
|---|---|
| `/`, `/about`, `/terms`, `/privacy`, `/author/alex-rivera` | Static (prerendered) |
| `/blog/[slug]` | SSG - one static page per MDX file via `generateStaticParams` |
| `/blog` | Dynamic (reads search params for filtering) |
| `/api/*`, `/feed.xml`, `/llms.txt`, `/sitemap.xml` | Server-rendered on demand |

## The blog (MDX) pipeline

Posts are plain `.mdx` files in `content/blog/`. The filename is the URL slug.

1. [src/lib/mdx.ts](../src/lib/mdx.ts) reads the files, parses frontmatter with `gray-matter`, and converts Markdown to HTML with `remark` + `remark-gfm` + `remark-html`.
2. [src/app/blog/[slug]/page.tsx](../src/app/blog/%5Bslug%5D/page.tsx) renders each post. It also:
   - injects heading IDs ([lib/heading-ids.ts](../src/lib/heading-ids.ts)) for the table of contents,
   - optimizes inline images ([lib/image-optimizer.ts](../src/lib/image-optimizer.ts)),
   - extracts FAQ Q&A pairs ([lib/faq-extractor.ts](../src/lib/faq-extractor.ts)) into FAQ structured data.
3. [src/components/BlogPostContent.tsx](../src/components/BlogPostContent.tsx) is the article layout (author, cover, TOC, related posts).

To add a post, drop in an MDX file + a matching cover SVG - no code changes. See [BLOG-INSTRUCTIONS.md](BLOG-INSTRUCTIONS.md).

## SEO & LLM discoverability layer

This site is built to be cited by AI search (ChatGPT, Claude, Perplexity, Google AI Overviews) as well as ranked traditionally.

| File | Output |
|---|---|
| [src/app/sitemap.ts](../src/app/sitemap.ts) | `/sitemap.xml` |
| [src/app/robots.ts](../src/app/robots.ts) | `/robots.txt` (explicitly allows AI crawlers) |
| [src/app/feed.xml/route.ts](../src/app/feed.xml/route.ts) | Full-text RSS feed |
| [src/app/llms.txt/route.ts](../src/app/llms.txt/route.ts) | `/llms.txt` - site summary for LLMs |
| [src/app/llms-full.txt/route.ts](../src/app/llms-full.txt/route.ts) | `/llms-full.txt` - full post text for LLMs |
| JSON-LD in each page | `Person`, `WebSite`, `BlogPosting`, `FAQPage`, `BreadcrumbList` |

All of these read the domain from `NEXT_PUBLIC_SITE_URL` and update automatically when posts are added. Page-level meta tags are generated by [src/lib/metadata.ts](../src/lib/metadata.ts).

## Contact / CTA

There is no newsletter or form backend. The primary call to action is a single **"Hire me"** button that opens a `mailto:` link, used in the navbar, hero, and final CTA section. Update the email address in those components and in the JSON-LD to make it yours.

## API routes

| Route | Purpose |
|---|---|
| `/api/indexnow` | Ping Bing/Yandex when content changes (daily Vercel cron) |

## Theming

Brand colors are CSS variables defined once in the `@theme` block of [src/app/globals.css](../src/app/globals.css):

- `--color-brand-*` → **purple** (primary)
- `--color-accent-*` → **pink** (CTA accent)

Tailwind 4 generates utilities (`bg-brand-500`, `text-accent-600`, …) from these tokens. Components never hardcode hex values. See [BRANDING.md](BRANDING.md).

> Note: blog cover SVGs in `public/images/blog/` hardcode their colors (SVG can't read CSS variables), so they must be updated by hand if the brand color changes.

## Config

- [next.config.ts](../next.config.ts) - security headers (CSP, HSTS, X-Frame-Options), and a `www → apex` redirect.
- [vercel.json](../vercel.json) - daily cron that calls `/api/indexnow`.
